# Prompt para Codex — Organizar `analysis/` e relatórios gerados

Você está trabalhando no repositório MDRG LuaLS addon.

## Problema

`analysis/` mistura vários tipos de documento:

```txt
- relatórios gerados por ferramentas;
- relatórios manuais;
- análises antigas;
- docs derivados de runtime probe;
- docs derivados de json dump;
- docs de callback/debug mod.
```

Exemplos:

```txt
analysis/lua_callback_contexts.md
analysis/realmod_asset_validation.md
analysis/mdrg_luadoc_comentarios_baseados_relatorios.md
analysis/chatgpt_mods_mixed_lua.md
analysis/relatorio_comparativo_mods_mixed.md
```

Sem cabeçalho de provenance, fica difícil saber:

```txt
- qual ferramenta gerou;
- quais inputs foram usados;
- se está stale;
- se pode ser apagado/regenerado.
```

## Objetivo

Adicionar política de relatórios gerados e provenance.

---

## Parte 1 — Classificar analysis files

Criar:

```txt
analysis/analysis_inventory.md
```

Tabela:

```md
| File | Generated by | Inputs | Regenerate command | Keep? | Notes |
|---|---|---|---|---|---|
```

Classificar cada `.md` e `.json` em `analysis/`.

Categorias:

```txt
generated-tool-output
manual-analysis
runtime-evidence
historical-analysis
schema
debug-mod-doc
candidate-archive
```

---

## Parte 2 — Cabeçalhos de provenance

Para relatórios gerados por tools, adicionar no topo:

```md
<!--
Generated by: tools/catalog_lua_callbacks.py
Inputs:
- library/*.d.lua
- luas/*.lua
- mod/realmod/script.lua
Regenerate:
python tools/catalog_lua_callbacks.py
Do not edit manually unless explicitly noted.
-->
```

Aplicar em outputs gerados:

```txt
analysis/lua_callback_contexts.md
analysis/callback_debug_cases.md
analysis/callback_debug_coverage.md
analysis/callback_debug_manual_test_steps.md
analysis/realmod_structure.md
analysis/realmod_asset_schema.md
analysis/realmod_item_patterns.md
analysis/realmod_asset_validation.md
analysis/debug_mod_real_asset_requirements.md
analysis/runtime_probe_stub_improvements.md
```

---

## Parte 3 — Mover análises antigas/volumosas

Propor mover para:

```txt
analysis/archive/
```

ou:

```txt
archive/analysis/
```

Candidatos prováveis:

```txt
analysis/chatgpt_mods_mixed_lua.md
analysis/mods_mixed_analysis.md
analysis/relatorio_comparativo_mods_mixed.md
```

Não apagar. Só mover se não forem usados por ferramentas.

---

## Parte 4 — Atualizar tools para escrever cabeçalho

Atualizar scripts geradores:

```txt
tools/catalog_lua_callbacks.py
tools/validate_realmod_assets.py
tools/validate_debug_mod_real.py
tools/probe_runtime_index.py
tools/generate_luals_addon.py
```

para incluir cabeçalho quando emitirem `.md`.

---

## Validação

Rodar ferramentas geradoras:

```powershell
python tools\catalog_lua_callbacks.py
python tools\validate_realmod_assets.py --root mod\realmod
python tools\validate_debug_mod_real.py --root examples\callback_debug_mod_real
python tools\probe_runtime_index.py
```

Se `mod/realmod` não existir no checkout, pular com aviso.

---

## Entrega esperada

1. `analysis/analysis_inventory.md`.
2. Provenance headers nos relatórios gerados.
3. Arquivos antigos movidos para archive se seguro.
4. README/AGENTS atualizado com política de analysis.
