# Prompt para Codex — Alinhar README/AGENTS/SKILL/.gitignore com o estado real do projeto

Você está trabalhando no repositório MDRG LuaLS addon.

## Problema

Há divergência entre a documentação e o estado real do projeto.

Exemplos:

- `README.md` diz que `luas/` é raw extraction local e ignorado por Git.
- `validate_addon.py` tem `RAW_TRACKING_DENYLIST = ["dump", "rawdump", "luas", "examples/mods_mixed.lua"]`.
- `.gitignore` ignora `dump/`, `rawdump/`, `examples/mods_mixed.lua`, mas **não ignora `luas/`**.
- O ZIP atual contém `luas/`, `probe/`, `json/mdrg_jsondump_tools/all_mods_dump/`, `archive/`, `old/`, etc.
- `TODO.md` marca fases completas, mas ainda é um TODO ativo na raiz.

## Objetivo

Criar uma política consistente de layout, tracking e packaging.
