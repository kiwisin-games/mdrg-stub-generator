# Callback debug manual test steps

Run only in a disposable test save. The debug mod is inert until flags in `script.lua` are enabled manually.

## Baseline

1. Set `ENABLE_DEBUG_ITEMS = true`.
2. Set `ENABLE_CALLBACK_PROBE = true`.
3. Keep optional groups disabled unless the specific scenario needs them.
4. Load the mod and check for `callback-report-*.txt` or console chunks.

## Item category matrix

Register all category items, then inspect item list/shop availability depending on enabled flags.

- `Clothing`: `category-clothing`
- `Food`: `category-food`
- `FoodCookedDish`: `category-food-cooked-dish`

## Runtime triggers

- Use a debug item to trigger `ItemPrefab.OnUse`.
- Unpack the package debug item to trigger `ItemPrefab.OnUnpack`.
- Eat food, fish, cooking ingredient or cooked dish debug items to trigger nutrition callbacks.
- Enable `DEBUG_ADD_TO_SHOP = true` and open the configured shop to trigger availability callbacks.
- Equip/render slotted debug items to give item-scope conditions and special effects a chance to run.
- Enable `DEBUG_ENABLE_TEXTURE_CALLBACK_CASES = true` only when validating texture override callbacks.
- Enable `DEBUG_ENABLE_STORY_CONDITION_CASES = true` to create story conditions; do not call `ConditionMet` manually from startup.
- Enable `DEBUG_ENABLE_COROUTINE_CASES = true` only when validating coroutine lifetime.

## Contextual or intentionally not automated

- `ItemPrefab.OnUnpack`: Runtime field is System.Func<Item, Item[]>; MoonSharp cannot convert a plain Lua function/table return into CLR Item[] without a C# bridge.
- `TextureOverriderManager.GetSpecialEffectsAction`: GetSpecialEffectsAction is an instance method; calling it on static TextureOverriderManager userdata fails.
- `ItemPrefab.SetCustomShader`: No shader asset is copied into this debug mod.
- `UiOverlay.*`: UI callbacks need explicit scene/UI interaction and should not be opened automatically.
- `ItemManager.*events`: Manager events require lifecycle ownership and unsubscribe policy before runtime use.
